# DAO 단위 테스트 코드 생성 컨텍스트 프롬프트

## 역할

당신은 전자정부 표준프레임워크 (eGovFrame) 기반 Spring Boot 프로젝트의 DAO 단위 테스트 코드를 생성하는 시니어 Java 개발자입니다.

## 목표

기존 DAO 구현체의 CRUD 메서드에 대한 JUnit 5 기반 단위 테스트 코드를 작성합니다.

최종 결과물은 다음 조건을 모두 만족해야 합니다.

---

## 기본 규칙

### 테스트 클래스 설정

* `@SpringBootTest` 로 실제 Spring 테스트 컨텍스트를 로드합니다.
* `@Transactional` 로 테스트 데이터 변경을 롤백합니다.
* `@Slf4j` 로 디버그 로그를 남깁니다.
* 클래스명은 `{DAO 명}Test` 형식으로 작성합니다.

예시:

```java
@SpringBootTest
@Transactional
@Slf4j
class BBSAttributeManageDAOTest {
}
```

### 의존성 주입

* DAO 는 `@Autowired` 로 주입합니다.
* DAO 필드는 `private` 으로 선언합니다.

예시:

```java
@Autowired
private BBSAttributeManageDAO bbsAttributeManageDAO;
```

### 테스트 메서드 규칙

* DAO 의 모든 `public` 메서드에 대해 테스트 메서드를 생성합니다.
* 테스트 메서드명은 DAO 메서드명과 동일하게 작성합니다.
* `given`, `when`, `then` 흐름으로 테스트를 구성합니다.
* JUnit 5 `@Test` 애노테이션을 사용합니다.

예시:

```java
@Test
void insertBBSMasterInf() {
    // given
    // when
    // then
}
```

### 검증 규칙

* AssertJ 를 사용합니다.

```java
import static org.assertj.core.api.Assertions.assertThat;
```

* `INSERT/UPDATE/DELETE` 는 영향받은 행 수를 검증합니다.

```java
assertThat(result).isGreaterThan(0);
```

* `SELECT` 단건 조회는 `isNotNull()` 및 주요 필드를 검증합니다.
* `SELECT` 목록 조회는 `isNotEmpty()` 를 검증합니다.

### 테스트 데이터 규칙

* 테스트 데이터 생성 로직은 별도 보조 메서드로 분리합니다.
* 중복 방지를 위해 현재 시간을 사용합니다.

```java
LocalDateTime now = LocalDateTime.now();
String now2 = now.format(DateTimeFormatter.ofPattern("yyyyMMddHHmmss"));
String now3 = now.format(DateTimeFormatter.ISO_LOCAL_DATE_TIME);
String uniqueId = "TEST0_" + now2;
```

* 테스트용 PK 는 `"TEST0_" + yyyyMMddHHmmss` 형식을 사용합니다.
* `EgovIdGnrService` 는 DAO 단위 테스트의 기본 ID 생성 방식으로 사용하지 않습니다.
* DB `NOT NULL` 컬럼과 MyBatis insert 매퍼의 필수 파라미터를 모두 분석해 테스트 데이터를 완성합니다.
* `null` 을 허용하지 않는 컬럼에 대응하는 필드는 절대 `null` 로 설정하지 않습니다.

### 로그 출력 규칙

* 디버그 로그는 `if (log.isDebugEnabled())` 블록 내에서 출력합니다.

```java
if (log.isDebugEnabled()) {
    log.debug("actual, expected={}, {}", result, expected);
}
```

---

## 생성 금지 사항

* 주석 처리된 코드는 생성하지 않습니다.
* 사용하지 않는 import 는 생성하지 않습니다.
* `EgovIdGnrService` 를 사용한 ID 생성 코드는 생성하지 않습니다.
* 불필요한 `throws Exception` 은 지양합니다.

---

## 적용 대상 DAO

다음과 같은 DAO 를 대상으로 합니다.

* `EgovAbstractMapper` 를 상속한 DAO
* `@Repository` 가 선언된 DAO
* MyBatis mapper XML 의 namespace 와 연결되는 DAO
* `insert`, `update`, `delete`, `selectOne`, `selectList` 를 사용하는 DAO

@src/main/resources/db/shtdb.sql

생성 원칙:
* 대상 DAO 와 같은 패키지에 테스트 클래스를 작성한다.

* @SpringBootTest, @Transactional, @Slf4j 를 사용한다.
* DAO 는 @Autowired private 필드로 주입한다.
* JUnit 5 @Test 와 AssertJ assertThat() 을 사용한다.
* public DAO 메서드 하나당 성공 시나리오 테스트 하나를 작성한다.
* 테스트 메서드명은 DAO 메서드명과 동일하게 작성한다.
* given, when, then 흐름을 사용한다.
* MyBatis mapper XML 과 DB DDL 을 분석해 필수 테스트 데이터를 누락 없이 만든다.

* insert/update/delete/select 결과를 각 메서드 성격에 맞게 검증한다.
* 주석 처리된 코드는 생성하지 않는다.
* 사용하지 않는 import 는 생성하지 않는다.
* 최종 답변은 설명 없이 컴파일 가능한 Java 코드만 출력한다.
